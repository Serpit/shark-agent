# talks/

教练咨询记录目录。由 `/talk` 命令触发，每次咨询产出一个文件。

## 文件命名

`YYYY-MM-DD-<topic-slug>.md`

- 同主题不同时间重新评估 → 开新文件（保留判断演变），新文件头部链接到旧文件
- 同一次 talk 跨多轮对话推进 → 同一文件（中途更新）
- 同一天同主题多次新 talk → `-1/-2` 后缀

## 文件模板（6 段）

```markdown
# <一句话问题摘要>

> 时间：YYYY-MM-DD HH:MM (UTC+8)
> 模式：问诊 / 体检
> 分类：信息问题 / 模式问题 / 心理卡点 / 假问题
> 关联实验：<experiments.md 中的实验名，无则写"无">

## 1. 问题
<用户原问题，保留原话>

## 2. 分类判定
<为什么归到信息/模式/心理/假问题，判定依据>

## 3. 公理扫描结果
<对照 axioms.md 9 条逐项扫描，触发哪几条，标注 ⚠️/⚡>

## 4. 关键追问
<agent 提出的关键追问和用户回答>

## 5. 消解/回答
<如果是假问题→消解（指出前提错误/给第三选项）；如果是真问题→给具体回答和行动路径>

## 6. 行动
<落到具体下一步，关联到 experiments.md 或 todos.md>

---
## 关联
- 前序 talk：[链接](YYYY-MM-DD-<slug>.md)（如有）
- 后续 talk：（待填）
```
