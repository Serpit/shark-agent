# 用户画像

阅读和评价文章时,按以下偏好排序权重:

- **主线**:沉淀**出海**方向的 SOP——出海相关的渠道、合规、获客、变现 SOP 优先级最高。
- **需求挖掘**:愿意从文章中反推未被满足的需求/市场缺口,而不只是看作者赚了多少钱。
- **试错容忍度**:能接受**国内小成本项目**作为体验和试错样本——国内小项目即使利润有限,只要能验证某种打法的可行性也值得保留。
- **人脉信号**:留意文章中提到的、值得建立连接的人或社群线索(作者、被采访对象、关键节点人物)。

## 硬约束

- **时间投入**:工作日每天约 2 小时,周末合计约 8 小时,总计约 18 小时/周。
- **试错预算**:未来 3 个月总预算上限约 1 万人民币。
- **投入方式**:必须先兼职启动;只有在跑出收入或明确正反馈后,才考虑全职投入。
- **回报周期**:希望 3 个月内看到有效反馈,不适合现金流周期过长、前期验证慢的方向。

## 能力与资源

- **核心能力**:技术开发。
- **当前职业背景**:在海外创业公司做 App 开发,对海外产品开发、上线流程、基础盈利模式有实际认知。
- **英文能力**:能阅读英文资料;配合 AI/翻译工具可以完成英文写作,但暂不默认具备高强度英文商务沟通或谈判能力。
- **资源现状**:暂无小语种、海外本地、供应链、外贸资源。

## 阶段目标

- **当前优先级**:个人 IP 优先,创业探索并行。短期不只追求找到一个产品方向,也要积累可被外界识别的专业表达、信任资产和机会入口。
- **3 个月目标**:在打造个人 IP 的同时,通过 MVP 或轻量验证找到真实付费用户;不追求一开始赚大钱,但要求商业逻辑合理、模式正确,不是靠偶然流量或一次性套利。
- **短期收入要求**:能覆盖服务器、API、工具等基础成本即可。
- **当前最优先验证能力**:找需求。先证明能发现真实痛点和付费意愿,再进入产品打磨。
- **收益偏好**:如果方向合法合规,且长期风险可控,用户更偏向选择赚钱潜力更高的方向,即使个人兴趣不是最强。

## 个人 IP 定位

- **主定位**:技术开发视角的海外创业探索。
- **内容支柱**:
  - 独立开发出海
  - 海外 App / 订阅商业化拆解
  - AI 提效与创业验证
- **表达原则**:个人 IP 是承接上述三条内容的外壳,不单独作为空泛主题展开;允许阶段性多题材输出,但最终要回到这三个支柱上。
- **希望传达的价值**:海外 App 开发经验、技术人创业探索、AI 提效开发、独立开发出海、普通程序员转个人品牌、海外产品商业化拆解;这些内容都可以作为关注理由。
- **目标受众**:普通开发者、迷茫的人、小白、想成功但还在焦虑的人。
- **平台优先级**:
  - `X`:主战场,发短推文 + 长文
  - `公众号`:用于锻炼长文能力
  - `小红书`:用 AI 产出做轻量尝试,偏实验性质
  - `YouTube`:想做持续变现的频道,目标是赚一点成本和启动资金
- **内容形式偏好**:
  - `X` 上适合短推文 + 长文
  - `公众号` 上适合长文
  - `小红书` 上适合简单、低成本、AI 辅助内容
  - `YouTube` 上希望逐步做成可变现频道
- **节奏偏好**:工作日做小内容和过程记录,周末做长文和总结;内容应尽量来源于创业探索过程本身,而不是脱离实践单独写作。

## 方向筛选权重

- **优先用户类型**:优先企业客户、开发者;其次是小团队或个体商家。
- **优先产品复杂度**:优先 1-2 周能做出 MVP 的小工具,便于兼职条件下快速验证需求与付费。
- **需求入口**:开发者自己的痛点、海外 App 用户痛点、企业内部效率痛点、小团队营销获客痛点都可作为入口;优先选择能快速接触真实用户和验证付费意愿的入口。
- **收费模式**:一次性付费、月订阅、按量计费、App 内购均可接受;长期维护型产品必须匹配订阅或持续收入。
- **维护接受度**:可以接受长期维护,但前提是有订阅收入或持续付费支撑,不能变成无偿客服和无限维护。
- **优先获客方式**:冷启动阶段优先 SEO、Reddit/社区、Product Hunt、App Store、Chrome Store、冷邮件、内容营销等免费或低成本渠道;付费投放可小预算测试,但不作为早期主要获客方式。
- **赛道偏好**:AI 工具、效率工具、开发者工具、营销工具、电商工具、创作者工具、教育工具、金融工具、健康工具均不预设排斥;但用户当前缺少互联网以外的垂直行业知识,进入强行业垂直方向前需要先补足 domain insight。
- **开发者工具态度**:可以尝试 Chrome 插件、CLI、API wrapper、调试工具、代码生成辅助、App 开发小工具等开发者工具,但必须有明确价值和付费理由。

## 偏好与禁区

- **销售/陌生沟通**:可以接受,但当前缺少真实场景训练;适合从低压力文字沟通、用户访谈、社区互动开始,逐步升级到语音/BD。
- **内容表达 / 个人 IP**:个人 IP 优先级高于短期产品探索。可以接受内容输出;露脸需要渐进式推进,早期更适合文字、录屏、半露脸或不露脸内容。内容方向应尽量服务长期信任资产,同时为创业探索提供需求观察、用户连接和反馈入口。
- **产品形态偏好**:工具型 SaaS、App、Chrome 插件、AI 自动化、内容站、模板数字产品、B2B 小工具、开发者工具都可接受,暂不强行限定。
- **明确不做**:灰产、擦边。
- **阶段性接受**:强销售和投放烧钱不是首选冷启动方式;如果产品已有验证、单元经济模型清楚,后续可以作为放大手段分阶段尝试。
