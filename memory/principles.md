# 理念库

跨方向通用的方法论与信念。区别于:

- `findings/`:单篇文章的提炼,**单点证据**。
- `themes.md`:具体的出海方向候选,**做什么**。
- `risks.md`:反模式,**不要做什么**。
- `principles.md`:**怎么思考、怎么判断**——为方向选择和动作决策提供准则。

每条理念由多篇 finding 的反复印证聚合而来,自下而上;反过来,选择 theme 或启动试错时,要回头对照 principles 是否一致。

## 模板

```
## <理念名,例如:先验需求再投产能>
- **陈述**:一句话讲清楚这个理念主张什么、反对什么
- **信心等级**:hypothesis / supported / strong   (随证据增加可升级)
- **形成依据**:[finding-链接](findings/xxx.md), [finding-链接](findings/yyy.md)
- **适用边界**:在什么场景下成立,什么场景下不适用
- **首次记录**:YYYY-MM-DD
- **最近更新**:YYYY-MM-DD
```

## 先发再迭代——做出 v0.1 优于完美计划

- **陈述**:哪怕第一版"糙得像垃圾"也要发出来,真实反馈是迭代的唯一燃料。"看攻略 1000 遍,不如做一次";同样适用于"创业 v0.1"——先做最小验证(例如一周内能不能拿到第一笔小钱)而非反复打磨方案。
- **信心等级**:supported(两条独立证据,产品层和创业层都印证)
- **形成依据**:[youtube-ai-faceless-channel-sop.md](findings/youtube-ai-faceless-channel-sop.md)(作者纯手工硬更 7 天才等到第一个万播,然后才动手做自动化);[2026-04-do-it-yourself-mindset.md](findings/2026-04-do-it-yourself-mindset.md)(作者建议朋友先 1 周内验证"出来干能不能拿到第一笔小钱")
- **适用边界**:在风险可控、单次失败成本低的场景成立;在涉及合规/品牌/重资金投入的场景需先做风控,不能盲发。
- **首次记录**:2026-04-27
- **最近更新**:2026-04-27

## 自动化前先核算时间成本

- **陈述**:做任何提效投入前,先把"每个工序当前消耗多少时间"量化,反推哪里值得自动化。时间是绝对稀缺,不算时间成本就分不清"看似在迭代实则在打杂"。
- **信心等级**:hypothesis
- **形成依据**:[youtube-ai-faceless-channel-sop.md](findings/youtube-ai-faceless-channel-sop.md)(作者用一整个月只做提效脚本 + 时间/经营成本核算,然后才扩号)
- **适用边界**:适用于已经跑通 MVP、要进入扩张期的项目;MVP 跑通前不要过度优化(否则陷入提前优化陷阱)。
- **首次记录**:2026-04-27
- **最近更新**:2026-04-27

## 卖解决方案,而非卖产品

- **陈述**:在出海/B2B 等成熟市场,只「报价 + 发货」已经被卷成红海;真正的溢价来自把产品打包成「解决方案」——产品 + 认证 + 本地化适配 + 服务/培训/远程支持,谈判时把价格谈判引导为价值确认。
- **信心等级**:hypothesis(单证据,但跨多种场景出现:小语种本地化 / 制造业配套 / 非洲枢纽都依赖这一思路)
- **形成依据**:[2026-04-foreign-trade-b2b-newbie-sop.md](findings/2026-04-foreign-trade-b2b-newbie-sop.md)(作者把三个机会窗口的核心都归结为「从中间商升级为解决方案提供商」,启动案例也是「灯具 + GCC 认证 + 伊斯兰美学设计 + 1 年远程支持」打包)
- **适用边界**:适用于客单价较高、决策链复杂的 B2B 出海;不适用于纯 C 端 SKU 站、流量型电商等比拼极致价格/物流的场景。
- **首次记录**:2026-04-28
- **最近更新**:2026-04-28

## 供应链优先于产品(B2B 出海特化)

- **陈述**:做 B2B 贸易,选品的起点不是「我要卖什么」,而是「我能稳定调动什么样的供应链」。先花主要精力筛选并绑定 1-2 家柔性工厂(支持小单快反多批次),再从工厂能力反推可做的产品赛道,而不是相反。
- **信心等级**:hypothesis(单证据)
- **形成依据**:[2026-04-foreign-trade-b2b-newbie-sop.md](findings/2026-04-foreign-trade-b2b-newbie-sop.md)(作者明确提出「评估供应链能力 → 锁定适配品类 → 验证市场匹配度 → 深化合作」的闭环,把工厂视为盟友而非低价比价对象)
- **适用边界**:适用于贸易公司/SOHO/小工厂代销模式;不适用于自有工厂/品牌方/数字产品/纯内容生意。
- **首次记录**:2026-04-28
- **最近更新**:2026-04-28

## 先验需求再投产能——BRD 优于 IDE

- **陈述**:打开 IDE 之前,先用 3 小时把"有没有人愿意为此付钱"调研清楚;独立开发者 90% 失败首因是没调研。AI 编程让 MVP 从 2 个月压到 2 周,但调研 AI 替不了你——**写代码是最贵的赌博,调研是最便宜的投资**。3 小时足以做 70% 调研,够做 GO/NO-GO 判断;NO-GO 不是失败,是省下 3 个月。
- **信心等级**:strong(两条独立证据,且与既有"先发再迭代"形成时间序补完——验证之后才是 v0.1)
- **形成依据**:[2026-04-indie-saas-3h-brd-validation.md](findings/2026-04-indie-saas-3h-brd-validation.md)(作者 pngtrid 先写 2 周代码上线才发现市场被占满,后改为 2 天调研重定位 1 个月变现 5K;给出 6 步 BRD SOP 与 GO/NO-GO 框架);[2026-04-foreign-trade-b2b-newbie-sop.md](findings/2026-04-foreign-trade-b2b-newbie-sop.md)(作者 19 年被女鞋营销话术裹挟未做调研,9 万启动金 3 个月烧光,触底负债 60w)
- **适用边界**:适用于商业方向选择/MVP 立项;**与"先发再迭代"互补不冲突**——本原则解决"该不该做",先发再迭代解决"决定做之后如何启动"。3 小时是首次破冰的数量级,完全陌生赛道实际需 1-2 天熟练后才能压到 3 小时
- **首次记录**:2026-04-28
- **最近更新**:2026-04-28

## 差评是金矿,3-4 星是产品需求文档

- **陈述**:做需求挖掘时,**用户怒火比用户夸赞值钱 100 倍**。不要看竞品官网,要看用户在哪里骂竞品。1 星看核心痛点,3-4 星看"差那一点"——那就是切入点;真痛点判定 = 同一痛点 ≥3 次 + 来自不同平台。"竞品知道问题但一直没改"就是你的护城河。
- **信心等级**:hypothesis(单证据但方法论极具体可执行,与"需求挖掘"画像权重高度对齐)
- **形成依据**:[2026-04-indie-saas-3h-brd-validation.md](findings/2026-04-indie-saas-3h-brd-validation.md)(给出三层平台 + 四路搜索 + 五步看法 SOP,案例从 G2/Reddit/Trustpilot 翻出 remove.bg 真实定价/额度痛点,30 分钟产出痛点频次表)
- **适用边界**:适用于成熟赛道(竞品已经存在,有公开评价数据);**不适用于全新品类**(没有竞品就没有差评可挖,得换成用户访谈/痛点观察)。要警惕单平台回音壁(Reddit 天然反付费),必须多平台交叉
- **首次记录**:2026-04-28
- **最近更新**:2026-04-28

## 不要裸辞创业——心态急是反模式

- **陈述**:在没有验证项目方向的情况下裸辞,会陷入「越急 → 越乱选 → 越亏 → 更急」的负循环;每个项目都现学现卖、1-2 个月浅尝辄止,本质是心态失衡而非方向错误。试错应在保留收入或现金流缓冲的前提下进行。
- **信心等级**:supported(作者两次创业期亲历,且明确以反例告诫;也与「自动化前先核算时间成本」形成同源警示——稀缺感会扭曲决策)
- **形成依据**:[2026-04-foreign-trade-b2b-newbie-sop.md](findings/2026-04-foreign-trade-b2b-newbie-sop.md)(作者从工作辞职后 SOHO 2.0 阶段「一直尝试很多项目,一直都不赚钱,换得很勤快」,亲承裸辞心态会比较急)
- **适用边界**:适用于个人/小团队创业启动决策;若已有充足现金流(覆盖 12+ 个月生活成本)且方向已小范围验证,则不构成约束。
- **首次记录**:2026-04-28
- **最近更新**:2026-04-28
