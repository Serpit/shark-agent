# 时间规划

## 阶段总览

- **2026-04 至 2026-07 上旬**:知识收集与沉淀阶段——目标是锁定出海方向、挖出具体市场。这个阶段优先广度与质量分类,不急于行动。
- **2026-07 起**:小规模试错阶段——从这之后,agent 的输出应更偏向「这个 SOP 我能不能马上跑一个最小验证」而非继续囤积信息。

## 双轨节奏(IP 优先,创业探索并行)

总盘:18 小时/周(工作日 2h × 5 + 周末 8h)。详细执行规则见 [experiments.md](experiments.md) 中的「个人 IP 定位与内容主线验证」实验。

### M1(2026-05):IP 起势 + BRD 第一轮

- **创业探索**:`mobile subscription app` 与 `expense tracker` 各 6h BRD 调研,出 GO/NO-GO 判断
- **IP**:X 累计 ≥12 条小推文 + ≥1 篇长文/总结;公众号 1-2 篇;小红书随手发
- **不启动**:YouTube
- **月底产出**:1 个 GO 候选 + 一个能解释清楚的 X 定位

### M2(2026-06):收敛 + MVP

- **创业探索**:GO 候选进入 1-2 周 MVP 编码 + 上线
- **IP**:X 内容自然切到「我在做 XX」实战叙事;月末启动 YouTube 第一支(用 MVP 复盘做素材)
- **公众号**:产出 1 篇 MVP 验证全流程长文

### M3(2026-07):冷启动 + 反馈循环

- **创业探索**:Reddit / Product Hunt / Chrome Store / App Store 冷启动,目标"覆盖服务器+API 成本"
- **IP**:进入实战复盘模式,转化率最高的阶段;评估是否带来 ≥3 条需求/用户线索
- **节奏切换判定**:同时回答 timeline 原本的"7 月切试错"问题——若 MVP 已上线、IP 已成型,即顺势进入试错阶段;若两者都没,触发下方降级路径

## 节奏提醒触发条件

- **M1 中段(每两周)**:若 X 周更新 < 5 条或连续 2 天断更 → 触发 IP 实验复盘,先砍小红书/公众号,保 X
- **M1 末**:若 X 累计 < 12 条 → 说明 18h 时间硬约束严重高估,要求重新校准 [profile.md](profile.md#硬约束)
- **M1 末**:若 BRD 第一轮 0 个 GO 候选 → 重新审视 [themes.md](themes.md) 主线方向是否需要重新定义,而不是继续加候选
- **临近 2026-07**:[experiments.md](experiments.md) 中没有任何实验通过 GO 进入 MVP → 主动提示用户:"是继续 BRD,还是先就现有候选启动 MVP 试错?"
- **进入 2026-07 之后**:若用户仍只投喂文章不启动试错 → 提示阶段已切换,应转入"可执行 SOP 优先"模式
