# 社群平台需求挖掘 SOP(小红书 / Reddit / X 通用)

> **定位**:从社群平台用户行为(搜索 / 收藏 / 评论)反推产品需求的可执行方法论。**面向出海主线**,小红书是国内变体,Reddit/X 是出海变体——同一套底层逻辑,平台特化点单独标注。
>
> **来源**:整合自 [`2026-04-wechat-miniprogram-ai-matrix.md`](../findings/2026-04-wechat-miniprogram-ai-matrix.md)("选需求三标准"+"看小红书评论区"零散动作)+ [`2026-04-indie-saas-3h-brd-validation.md`](../findings/2026-04-indie-saas-3h-brd-validation.md)(差评是金矿 + 6 步 BRD 法)+ 本人在教练对话中的整合推导(2026-04-29)。**文章本身没给完整 SOP,本文是我整合扩展的版本,使用时按需调整。**

## 底层逻辑

社群平台机制让用户**主动反复搜、收藏、评论**,留下三类**可量化的需求信号**:

| 信号源 | 含义 | 平台对应 |
|---|---|---|
| **搜索热词** | 用户主动找答案 | 小红书搜索栏 / Reddit 站内搜索 / Google 带 site:reddit.com |
| **高收藏笔记** | 用户认可"我要留着用",**工具型需求强信号** | 小红书收藏数 / Reddit "save" / X bookmark |
| **评论区追问** | 现有内容/产品没满足 | 小红书评论"求链接""怎么用" / Reddit "is there a tool that..." / X 回复 |

**关键判别**:**收藏 > 点赞 = 工具型需求**(用户想留着用);**点赞 > 收藏 = 看个乐内容**(不是产品需求)。Reddit 等价物是 **save / upvote 比**。

## 6 步 SOP(每个候选约 1 小时)

### Step 1 · 选 3-5 个候选场景(15 分钟)

**不凭空想**,从你**有触感的场景**出发:
- 你过去 1 个月在小红书 / Reddit 自己搜过的问题
- 身边人(同事/家人/海外同事)反复抱怨的事
- 你自己工作/生活中有重复动作但没好工具的事

**反例**:凭空想"AI 教育有没有需求" — 太宏大无法挖。

**输出**:3-5 个具体场景关键词,例如"亲子博物馆讲解""跨境购物比价""Notion 模板批量生成"。

### Step 2 · 三维探针(每个候选 15 分钟)

| 动作 | 看什么 | 判定阈值 |
|---|---|---|
| 搜关键词 | 笔记/帖子总数 + 最新 30 天发布频率 | 总数 > 1000 + 30 天活跃 = 真实需求 |
| 看头部笔记/帖子 | **收藏/点赞比 + 收藏绝对值**(Reddit 用 save/upvote 比) | 收藏 > 点赞 = 工具型;单篇收藏 > 1k 多篇 = 强信号 |
| 看评论区 | 反复出现的问题 | "求链接""怎么用""有没有 App""is there a tool" — 都是产品空白信号 |

### Step 3 · 验证"头部产品做得烂"(10 分钟)

需求真实之后看供给质量。

| 国内 | 出海 |
|---|---|
| 微信搜小程序 / 应用市场搜 App | Chrome Store / Product Hunt / App Store / G2 / Capterra / Trustpilot |

**判定**:
- 现有产品丑/卡/广告多 → **机会**
- 大厂占据(随手记 / Notion / Calendly 级别)→ pass
- 完全没现有产品 → 进一步验证(可能没需求,也可能真空白)

### Step 4 · 抓 1 条"具象痛点"(10 分钟)

**从笼统需求到产品需求文档**的桥:

- ❌ "这个 App 不好用" — 太笼统
- ✅ "为什么没有一个能离线、无广告、按博物馆分类的讲解工具" — 这就是 PRD

**至少看 5 条头部笔记/帖子的评论区**,同一痛点出现 **≥3 次** 才算真信号(对齐「差评是金矿」原则的"≥3 次 + 不同平台")。找不到 → 这个方向还在"需求模糊"阶段,跳过。

### Step 5 · 套"选需求三标准"做过滤

| 标准 | 含义 | 反例 |
|---|---|---|
| 功能单一 | 一个产品只解决一个问题 | 全家桶工具 |
| 场景清晰 | 一句话描述用户什么时候打开 | "效率工具"(描述不出场景) |
| 没有头部产品 | 大厂没碰或做得烂 | 通用记账(随手记已饱和) |

三标准全过 → Step 6;任一不过 → pass。

### Step 6 · 1 页 BRD(对接 [钦哥 6 步法](../findings/2026-04-indie-saas-3h-brd-validation.md))

到这步与已有 BRD SOP 对齐,3 小时内产出:

- 竞品表
- 用户画像(从评论区扒昵称/年龄/地域)
- 痛点频次表
- MVP 范围
- 收费方式
- 首批获客渠道

## 平移对照表(国内 ↔ 出海)

| 步骤 | 国内(小红书 + 微信) | 出海(Reddit + X + Chrome Store) |
|---|---|---|
| 候选场景 | 你刷小红书搜过什么 | 你 / 海外同事 / 国际社区抱怨过什么 |
| 三维探针 | 笔记数 + 收藏 + 评论区追问 | subreddit 订阅数 + upvote + 评论里 "is there a tool that..." |
| 供给验证 | 微信搜小程序 | Chrome Store / Product Hunt / G2 / Trustpilot |
| 具象痛点 | 小红书评论 | Reddit 1 星评论 / r/SaaS / r/Entrepreneur 的 "I wish there was..." 帖 |
| 三标准 | 同上 | 同上 |
| BRD | 同上 | 同上 |

## 4 个避坑(平台特化)

1. **"评论区随便有人问"≠ 需求** — 至少 5 条头部笔记/帖子的评论区,**同一痛点 ≥3 次**才算
2. **小红书人群偏女性 / 一二线 / 年轻** — 候选若是"中老年 / 男性 / 三四线"需要换抖音 / 知乎 / 公众号 SEO 验证
3. **算法回音壁** — 你越搜某词推荐越多,容易过度乐观。**用无痕窗口或别人账号**做探针(Reddit 同样适用——多账号 / 退出登录)
4. **Reddit 天然反付费** — 在 r/SaaS、r/Entrepreneur 看到的"用户都不愿付费"很可能是单平台回音壁,**必须交叉 Product Hunt / X / Trustpilot 验证**(对齐「差评是金矿」原则的"多平台交叉")

## 何时使用

- 启动新方向 BRD 之前(Step 1-5 是 BRD 的"找需求"前置)
- 已有 BRD 但想交叉验证需求强度时(单平台跑一遍)
- 个人 IP 选题时——同样可用来挖**内容选题**而不只是产品(用户在反复问什么 = 你写什么有人看)

## 何时不适用

- 全新品类(没有现成竞品也没有用户讨论 — 需要换成用户访谈)
- 强行业垂直方向(行业内幕在小红书/Reddit 显示不出来 — 需要 LinkedIn / 行业垂直社区)
- 需要监管资质的方向(医疗 / 金融 — 仅靠社群信号不足以判断)
