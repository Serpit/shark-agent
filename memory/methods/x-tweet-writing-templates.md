# X 推文起草 SOP(模板 + 自检)

> **定位**:M1 内容输出节奏的起草层——把"今天发什么"上升到"今天用哪个结构表达哪条判断"。每条推文从这套 SOP 走,目标:把 60 分推文稳定提到 80 分,80→95 靠用户自己的经历/判断/语气。
>
> **来源**:[2026-04-x-tweet-writing-templates.md](../findings/2026-04-x-tweet-writing-templates.md)(熊老板/Mr. Panda)+ 本项目的"换名字还成立吗"自检接入。
>
> **何时使用**:
> - 起草任何一条 X 推文/Thread 之前(strict)
> - 复盘已发推文时回头打分
> - todos.md「待发推文」表里每条标"目标结构"

---

## 底层原则(开始套模板前先记住)

1. **第一行是唯一战场**:1 秒判定停留 / 滑走,展开前可见 20-40 字。把最有冲击力的信息放第一行,**不要预热**。
2. **质量在选题时决定,不在写作时**:今天没有值得发的判断/经历/观察 → 直接发"今天的卡点"或不发,**不要为发内容反向编选题**(对应 [experiments.md 选题铁律](../experiments.md#个人-ip-定位与内容主线验证))。
3. **不可替代性 > 结构正确性**:模板是骨架,**不是替代你独有的细节、判断、经历**。每条都通过"换名字还成立吗"自检。

## 起草 SOP(单条推文,约 5-15 分钟)

### Step 1 · 锁定要表达的判断/经历/观察(2 分钟)

回答一句话:**这条推文要让读者带走什么具体的东西?**

- 一个反共识判断
- 一个我做过/没做过的具体事 + 教训
- 一个我刚提炼出的方法/框架
- 一个我观察到的现象 + 解释
- 一段过程进展 + 数据

**如果回答是"觉得这个话题流行"或"应该写点什么"——pass,今天换选题或不发**。

### Step 2 · 选目标结构(2 分钟)

按表 A 选 1 个开头型 + 1 个推文结构(单条),或表 B 选 1 个 Thread 骨架。**默认从该结构最匹配的触发场景倒查**:

#### 表 A · 14 个单条结构(4 开头 + 10 主体)

**4 种开头型(写完主体后专门改第一行用)**:

| 开头型 | 公式 | 你今天的表达适合吗 |
|---|---|---|
| 反差型 | 大多数人以为 A,实际上 B | 你有反共识判断 |
| 数字锚定型 | 具体数字 + 你做了什么 + 悬念 | 有可量化的过程/结果 |
| 悬念型 | 戏剧性事件 + 留白 | 有真实经历转折 |
| 直接宣言型 | 一个斩钉截铁的判断句 | 有强观点,不想铺垫 |

**10 种主体结构(选一个匹配你今天判断/经历的)**:

| # | 结构 | 触发场景(你今天的输入) | 主体节奏 |
|---|---|---|---|
| 1 | 认知颠覆 | 反共识 + 论据 | 挑战常识 → 点问题 → 论据 → 独立可传播结论 |
| 2 | 个人经历拆解 | 做成/失败一件事 | 结果数字前置 → 时间线 → 3 个关键转折 → 1 条核心经验 |
| 3 | 框架输出 | 总结成思维模型 | 痛点 → 框架名 → 3-5 要素 → 使用边界 |
| 4 | 清单 | 高密度实用信息 | 痛点 + 承诺 → 每条带推荐理由 → 个人首选 |
| 5 | Before/After | 状态转变 | Before 数字 → After 数字 → 改了哪些行为 → 反直觉认知 |
| 6 | 过程透明 | 正在做、有进展 | 阶段 → 数字进展 → 意外发现 → 下一步 |
| 7 | 对比分析 | 两个常被比较的事 | 提对比 → 多维度差异 → 选择条件 → 限定边界 |
| 8 | 故事钩 | 有转折的真实经历 | 起点(悬念) → 多个转折 → 高潮 → 一句话提炼 |
| 9 | 教学实操 | 教读者一个具体方法 | 痛点 + 预期结果 → 步骤(每步一句) → 注意事项 |
| 10 | 观点宣言 | 有争议立场 | 立场 → 论据 → 边界条件 → 强化句 |

#### 表 B · 3 种 Thread 骨架(长内容)

| 类型 | 骨架要点 | 触发场景 |
|---|---|---|
| 拆解型 | 钩子(数字/成本/结果) → 经验承诺 → 每条 1 经验 → 反直觉那条单列 → CTA | 长期做某事的复盘 |
| 故事型 | 起点悬念 → 转折(每条留钩) → 高潮 → 提炼认知 → CTA | 有起伏的真实经历 |
| 教程型 | 痛点 + 教什么 + 完成结果 → 前提 → 步骤 → 坑点 → "保存这条 Thread" | 教会读者完成一项任务 |

**Thread 铁律**:每一条都要能独立存在。读者从任何一条进入都不应被上下文卡住。

### Step 3 · 按结构填内容(5-10 分钟)

按选定结构的节奏填,**不一定一气呵成**——可以先把主体写完,最后专门改第一行。

填入时强制塞入两类**只有你能提供的颗粒度**:
- **细节**:数字、时间、地点、具体场景(不是"我做了 X",是"4 月 28 日凌晨 1 点跟教练把方向推翻第 3 次")
- **判断**:你敢辩护的那个观点(含糊的"也有道理"删掉)

### Step 4 · 改第一行(2 分钟,strict)

主体写完,**回头单独打磨第一行**:

1. 删掉原第一行(大多数人的第一行是"预热",读者不需要预热)
2. 找到主体里**最有冲击力的那句话**,移到最前面
3. 改成判断句——去掉"我觉得""可能""也许"
4. 朗读一遍。如果你自己都没有想继续往下读的冲动,再改

**检验标准**:好的开头让人产生"等等,真的吗?"的反应。

### Step 5 · 发布前三问(1 分钟,strict)

| 问 | 通过标准 |
|---|---|
| 1. 第一行有没有让人停下来的冲击力? | 朗读时自己有"等等"的反应 |
| 2. 读完之后,读者能带走一个具体的东西吗? | 能用一句话复述,不是"挺好的" |
| 3. **如果把署名换成别人,这条推文还能成立吗?** | **不能成立才合格**;能成立 → 回去加一个只有你知道的细节,加一个只有你才会下的判断 |

**任一不通过 → 回 Step 3 重写**。第 3 问最常出问题,也是 80→95 的唯一杠杆。

## 发完之后(回流到 todos.md)

1. 在 [todos.md 本周期已发](../todos.md#本周期已发滚动) 表里加一行,记录:日期 / 平台 / 主题 / 链接 / **使用的结构** / 互动数据
2. 进度计数 +1(X N/12)
3. 如果有意外反馈(被大号引用 / 评论里带出真实需求线索) → 单独标注,可能是下次复盘"哪个结构在我账号上有效"的素材

## 反模式与软提示

- **不要连续 3 条跑同一个结构**——读者会形成"又是套路"的反感。混用单条 + Thread,混用宣言型 + 经历拆解。
- **优先级倒错警示**:不要在没选题时套结构催稿;先有"今天值得发的判断/经历/观察",再选结构。**没有判断的模板是空架子**。
- **公理 6 心理卡点提示**(本项目特化):
  - 结构 8 故事钩 / 结构 5 Before-After / 结构 6 过程透明 都需要暴露具体经历
  - 如果连续 1-2 周只写认知颠覆型/清单型/观点宣言型 → **大概率是回避暴露,而不是结构选择**
  - 触发时回头追问:"是没有故事可写,还是不敢写出来?" 见 [axioms.md 公理 6](../axioms.md)
- **第一行优化的天花板**:再好的钩子也救不了没有信任价值的选题。**互动持续低 ≠ 钩子问题**,大概率是选题本身或不可替代性问题——回到第 3 问。
