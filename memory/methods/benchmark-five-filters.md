# 五重过滤（benchmark）

把 themes.md 候选方向做结构化对标,五重过滤逐项筛。来源：dbskill `/dbs-benchmark` 核心框架,适配为 shark-agent 的 themes 候选评估工具。

## 五重过滤

按顺序过,任一层不通过即可淘汰候选,不必跑完全部五层。

### 1. 同模式

**问句**：有没有人用同一种商业模式（相同的获客方式 × 收费方式 × 交付方式 × 收入结构）已经跑通?

**一句话定义**：不只找"做类似产品的人",要找"用同样方式赚钱的人"。

**示例**：评估一个 Chrome Extension 订阅制产品 → 对标也必须是"Chrome Extension + 订阅制",不能拿"Web SaaS + 订阅制"当对标（分发和获客结构不同）。

### 2. 同阶段

**问句**：对标对象处于什么阶段?你当前处在什么阶段?阶段差多远?

**一句话定义**：用 M5 阶段的对标衡量 M1 阶段的自己,等于用马拉松选手的配速要求一个刚会走的人。

**示例**：某 SaaS 年入 $100K 是"有邮件列表 5 万 + 3 年 SEO 积累 + 团队 3 人"的结果,你刚注册域名就拿它当收益预期 → 不通过。

### 3. 同合规

**问句**：对标案例的合规环境和你一样吗?它有没有你没注意到的牌照/资质/地区优势?

**一句话定义**：同一套打法在不同合规环境下的合法性和可行性不同。

**示例**：美国 LLC + Stripe 的 SaaS 定价策略不能直接复刻到"中国个人 + 无 Stripe"场景——收款链路是合规的一部分,缺失时整条线的单元经济不成立。

### 4. 同需求层

**问句**：对标产品满足的需求和你瞄准的需求是同一层吗?不是"看起来相关"而是"用户付费动机相同"?

**一句话定义**：相似的表面功能可能服务不同需求层（止痛药 vs 维生素 vs 化妆品）。

**示例**：Notion 模板"第二大脑"卖的是"我变得更高效"的自我认知（化妆品层）,而一个项目管理 SaaS 卖的是"老板看到交付进度"（止痛药层）。不能因为都和"效率"相关就做对标。

### 5. 同验证

**问句**：对标案例公开的数据中,有哪些是你无法复刻的验证（Product Hunt 当日排名、媒体报道、关键人背书、已有受众）?去掉这些后剩下的增长是你的吗?

**一句话定义**：把对方的一次性 boost 因子剥离,剩下可复刻的系统性增长才值得参考。

**示例**：某产品"上线 3 天 1000 付费用户"是因为 Product Hunt #1 + TechCrunch 报道 + 作者 Twitter 5 万粉。去掉这三个 boost,真实自然增长可能是"3 个月 50 付费用户"——这才是你的对标基线。

## 使用方式

评估 themes.md 候选时手动调用,不硬性要求 themes.md 模板包含"过滤记录"段。只在新增 theme 候选时跑一遍五重过滤,通过的标注 `benchmark ✓`,不通过标注淘汰原因。
