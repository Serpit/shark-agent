# 独立开发者出海 SaaS 的 3 小时 BRD 调研法

- **摄入时间**:2026-04-28
- **raw 文件**:`raw/我花3小时调研了一个出海产品，发现90%的独立开发者第一步就走错了.md`

## 价值标签

- 出海 SaaS / 独立开发者 / 需求挖掘 SOP
- 写代码前的需求验证框架,把"3 个月弯路"压缩成"3 小时调研"
- 用具体工具 + 具体关键词 + 具体打分卡的可执行手册,不是讲理论
- 跨方向通用——抠图工具只是案例,框架适用于任何出海 SaaS / 工具型产品

## 核心 SOP 与原理

### 顶层心法

1. **调研不是可选项,是生存技能。3 小时调研 > 3 个月弯路**——独立开发者只有一个人,没有试错的资本。AI 编程让 MVP 从 2 个月压到 2 周,但**调研 AI 替不了你**。
2. **70% 就动手,剩下 30% 市场会教你**——卡在调研阶段调研 3 周不动手,与不调研盲冲一样危险。给自己设 3 小时 deadline,到点必须做 GO/NO-GO 判断。
3. **调研的真正价值不是结论,是过程**——亲手翻 200 条评论 + 算市场数据 + 直接问 20 个用户后形成的市场理解,任何报告都换不来,会在后续 3 个月甚至 3 年持续给你回报。

### 6 步 BRD(Business Requirements Document)流程

| 步骤 | 时长 | 输出 | 关键工具 |
|---|---|---|---|
| 1. 趋势验证 | 5 分钟 | "是不是真需求 / 持续刚需" | Google Trends |
| 2. 用户骂竞品 | 60 分钟 | 痛点频次表 | G2 / Capterra / Trustpilot / Reddit / Twitter / HN / Chrome 商店 |
| 3. 用户画像 | 20 分钟 | 一个具体的人 + 付费意愿 | Reddit 潜水 + 直接发帖问 |
| 4. 市场规模 | 20 分钟 | TAM / SAM / SOM + 收入反推 | SimilarWeb + Statista + Crunchbase |
| 5. 定价 | 20 分钟 | 中位价方案 + Freemium 校准 | 列竞品定价表 + 社群验证 |
| 6. 算账决策 | 15 分钟 | GO / NO-GO 打分 | 决策框架 |

#### Step 1:Google Trends 一分钟筛真需求

- **平稳上升 = 持续刚需**;暴涨暴跌 = 接盘陷阱(典型反例:ChatGPT 刚出时"AI writing tool"3 个月涨 500%,半年跌回原点)。
- 时间拉到 5 年,排除疫情等临时因素;看搜索量最大地区,确认主战场语言。
- **关键技巧**:用更窄的关键词("background removal"→"background removal from product photo")发现细分场景增速,再点"Related Queries"找增速最快的小赛道(案例:"bulk background removal"指数低但同比 +48%,小群体高付费意愿)。
- 趋势下降不等于不能做,但要弄清"为什么下降"——是市场在萎缩(别碰),还是被系统功能替代(找内置方案没做好的部分)?反例:在线 PDF 压缩被 Chrome / macOS 内置功能替代,几乎不可逆。

#### Step 2:60 分钟看用户在哪里骂竞品(全篇最高 ROI)

**核心原则**:不要去看竞品官网,**去看用户在骂什么**。一个 3 星用户说"还不错但太贵了",比 100 个 5 星好评有价值——3 星用户差的那 2 星,就是你的机会。

**三层平台**:
- **主流评价**:G2 / Capterra(企业评价细致)、Trustpilot(普通用户情绪化)、Product Hunt(早期反馈尖锐)
- **社区(情绪最真实)**:Reddit 搜"[竞品] sucks/overpriced/alternative"、Twitter "[竞品] frustrated"、HN Show HN 评论
- **应用商店(最被忽视)**:Chrome 扩展 1-3 星评论 / App Store / Google Play 差评区是金矿

**四路搜索**(覆盖用户从"发现问题"到"准备付费"全过程):
1. **求助类**:"how to do XX" — 找正在找解决方案的人;前 10 个结果若 5 个以上是教程/问答(非竞品官网),说明商业化未完全覆盖,DIY 用户很多,有迁移机会
2. **吐槽类**:"[竞品] sucks / overpriced / alternative" — 差评是最值钱的数据;**评论区比标题有价值 10 倍**
3. **替代类**:"[竞品A] vs [竞品B]"、"switched from X to Y" — switched from 是金矿(用户具备付费能力 + 痛点 + 决策能力,正是你的目标)
4. **交易类**:"best XX 2025"、"XX pricing" — 找已经准备付费的人

**5 步看法 SOP**:
- 1-2 星差评(5 分钟):用 Ctrl+F 搜过滤词(expensive/slow/buggy/frustrating)统计频次,**看 10 条以上找共同点**
- 看竞品互拉(5 分钟):"switched from"语境
- 3-4 星评论(10 分钟):用户写得最长 + 最有建设性,**就是你的产品需求文档**
- 看时间线(5 分钟):**只看最近 3-6 个月**,老评论可能竞品已修
- 看用户身份(5 分钟):统计哪种身份出现最多,那就是你的核心用户群

**真痛点判定**:同一痛点出现 ≥3 次 + 来自不同平台才算真的;1-2 次是噪音。**频次最高的不一定最好**——"太贵"被说最多说明竞品自己也知道,你要找"频次不低 + 竞品一直没解决好"的痛点(护城河)。

**两个常见误区**:
- 只看一个平台 → 看到的是回音壁(Reddit 天然偏反付费);**多平台交叉验证**
- 只看差评数量不看内容 → Trustpilot 2.3 分不等于产品差,可能 1 星全是"信用卡被扣""账号被锁"等客服问题(你也未必能做更好);**关键是"差评里有多少是你可以解决的"**

#### Step 3:具体的用户画像(必须含付费意愿)

- "需要 XX 的所有人" = 没有画像。需要一个具体的人(姓名 + 职业 + 当前方案 + 月花费 + 最大痛点 + 付费意愿)
- **不愿付费的画像 = 精心包装的自欺欺人**;画像必须答清"付多少 / 多久付一次 / 现在付给谁"
- 直接去目标社群发帖问("现在付多少 / 愿意付多少"),案例:Reddit 23 个回复 → 82% 愿意付 $10-30/月,中位 $15;比作者自己猜的 $5-10 高 50%(按猜测定价就少赚了)
- 画像是迭代的,通常 2-3 轮才靠谱;画完去社群验证,与真实讨论不吻合就改

#### Step 4:市场规模 TAM / SAM / SOM

- **从上往下**:Google 搜"[品类] software market size",找行业报告(2025 抠图软件 $14.5 亿,CAGR 8%);**两份报告差不超 30% 才可信**,再看上市公司财报交叉验证(Adobe、Canva 财报里 AI 抠图相关条目)
- **从下往上**:SimilarWeb 查竞品月访问量(remove.bg = 3000 万/月)→ 工具型 SaaS 付费转化率保守取 0.1% → 反推月付费用户和收入
- **第三验证**:Crunchbase 查融资额,VC 估值常按年收入 5-10 倍,反推年收入(remove.bg 母公司 Kaleido AI 2021 融 A 轮 $1200 万 → 估算年收入 $240 万,与 SimilarWeb 反推同量级)
- **SOM 倒推法**(比拍脑袋准):你的获客渠道每天能带来多少新用户 × 注册→付费转化率(新站保守 1%) × 12 个月。案例算出 $10,800-16,200/年,比"500-1000 用户"的乐观估算低很多但更接近现实。**前 6 个月不赚钱是正常计划,不是意外。**

#### Step 5:定价用中位数 + 锚定,不用最低价

**为什么低价是陷阱**(三个理由):
1. 你的成本结构和大厂不一样——大厂规模摊薄后单张 $0.001,你 $0.01,你卖 $5 它卖 $9,毛利率反而它高
2. 低价吸引来的不是付费用户是薅羊毛的——案例:从 $9 降到 $5,用户量 +40% 但付费转化率 2%→0.8%,总收入反而 -15%
3. 低价锁死未来涨价空间——一开始 $5 半年后涨 $9 老用户骂;一开始 $12 后来促销到 $9 用户反而觉得占了便宜

**正确做法**:
- 列所有竞品定价表,**取中位数,不要取最低**
- **锚定效应**:用户看到 remove.bg $39/月这个"锚",再看你 $9/月觉得"合理便宜";若你定 $3/月反而觉得"质量不行"。**不做最便宜,做中偏上;用户选的不是便宜,是划算**
- **Freemium 校准**:免费版额度让用户**大约每周用完一次**——太少用户感受不到质量,太多用户没升级动力。10 张/月对每天 50 张的用户刚好"好用但不够用"
- **价位心理实验**:同社群 A/B 调查,$5/月100张 vs $9/月200张 → 选 B 多 37%(单价更低 + 200 张看起来"更够用")
- **提问技巧**:不要问"愿意付多少"(会往低说),问"你现在每月花多少"(透露已有支出 = 心理锚点)

#### Step 6:GO / NO-GO 决策框架

**GO 条件(满足 3 条就做)**:
1. ≥3 个不同平台验证的真实痛点
2. 明确差异化切入点(竞品没做好的)
3. TAM 增长(Google Trends 上行)
4. 启动成本可控(<$500)
5. 6 个月内可以回本

**NO-GO 信号(满足 2 条就不做)**:
1. 找不到 3 个真实痛点
2. 竞品已经做得很好且免费
3. 市场在萎缩
4. 启动成本超出承受能力
5. 没有任何获客渠道

**NO-GO 不是失败**——花 3 小时发现不值得做,比花 3 个月做完才发现要聪明 100 倍。**最贵的 NO-GO**:一个朋友花 4 个月做在线简历生成器,Canva 已免费 + Zety 占 SEO,4 个月 0 收入,连试都没试。

### 关键认知

- **大厂的战场是 Google 首页,你的战场是 Reddit 第 3 页**——不走大搜走垂直渠道(本案例:Reddit 电商社区 → Shopify 插件市场 → YouTube 教程,按月分阶段启动,不同时铺)
- **大厂做了不代表做好了**——Canva 免费抠图边缘粗糙,Photoshop $23/月对电商卖家太贵。大厂服务"80% 够用",你服务"剩下 20% 要满意"——这 20% 是最愿付费的人(每天处理 50+ 张的电商卖家)
- **最大风险不是技术,是获客**——技术有 100 种方案,获客只有"找到一条路"且随时可能堵死
- **MVP 思维:先用别人的轮子跑通商业模型,再用自己的技术降本**——不要一上来就追求技术自研(除非技术就是核心竞争力);案例:先用 Clipdrop API($0.02/张)上线,有付费用户再迁移到自部署 RMBG-1.4($0.005/张)

## 风险提示

- **作者本人案例待打样**:文中 6 步是用"在线抠图工具"做的演示推演,不是已上线产品的复盘;作者说"GO"但并未给出该产品上线后的真实数据,只能验证"调研框架本身可执行",不能直接迁移结论
- **3 小时神话需谨慎**:对完全没接触过的赛道,3 小时只够"破冰"——平台账号注册、术语扫盲、社区潜水都不止 3 小时;实际首次跑通需要 1-2 天,熟练后才能压到 3 小时
- **小语种 / 非英语市场不适用**:整套方法论高度依赖英语关键词搜索 + 英语平台评论(G2 / Reddit / Trustpilot / HN),做日韩 / 中东 / 西语 / 东南亚 SaaS 时需自行替换信息源(Naver / Yahoo Japan / 阿语 forums / VK 等),没有现成 SOP
- **付费转化率 0.1% 的拍脑袋**:作者算 remove.bg 月入用了"工具型 SaaS 取 0.1%"——这是行业经验值不是观察值,数量级估算可以,精确到月入 $27 万就过度了;**用市场规模做决策,不要用市场规模做财务模型**
- **"切入垂直渠道就能赢"**:Reddit / Shopify 插件 / YouTube 三个渠道也不是 free lunch——Reddit 反广告气氛极强,先 2 周纯回答问题做信任成本不低;Shopify 插件市场已饱和,新插件冷启动同样难;YouTube 教程内容生产成本高
- **"AI 编程 2 周做完 MVP"叙事**:文中默认"代码不是问题",但 0 经验的独立开发者用 AI 编程从需求到上线远不止 2 周——容易低估"调好部署 / 接支付 / 反诈合规 / 客服"等隐藏成本
