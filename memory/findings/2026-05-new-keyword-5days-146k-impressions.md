## 核心 SOP / 原理

### A. 选词信号源:Reddit 爆贴 + Trends 双重验证

| 步骤 | 动作 | 判定 |
|---|---|---|
| 1 | Reddit sub 闲逛,看到**评论数突然爆掉**的帖子(几百评论) | 评论数本身就是需求信号 |
| 2 | 提取帖子背后的关键词(工具名 / 现象名 / 需求名) | 不是看作者夸什么,而是看用户在评论里问什么 |
| 3 | 关键词丢进 Google Trends | **曲线从 0 拐头向上 = 强信号** |
| 4 | 注册域名 | 即使前 2 次失败,也按这个流程下注 |

**关键缺口(本项目今日实战补充,2026-05-01)**:作者**没有明确做 Ahrefs 绝对月搜索量验证**——只看了 Trends 的相对曲线。本项目用户在 `GPT Image 2` 候选词上踩坑:Trends 4-22 峰值 100 → 5-01 衰减到 28(看似"过窗口期但还有热度"),Ahrefs 一查 12 个变体全部 <100/月——**Trends 上升的相对曲线 + 父级搜索量 <100/月 = 0 流量**。
**结论**:Reddit + Trends 是**采购信号**(告诉你某个词有用户在讨论 / 在搜索增长),Ahrefs 绝对月搜索量是**底线判定**(告诉你这个词撑不撑得起一个站),缺一会跑死。SOP 红线"<200/月直接 pass"独立于其他信号。
**对作者前 2 次失败的反推**:作者自己说"前两次词起来一半就熄火",大概率就是 Trends 看到上升曲线 + 没做 Ahrefs 绝对值校验,实际搜索量基数太小。

### B. 写代码前的需求挖掘(SEO 站特化)

```
1. 锁定爆贴(Reddit 评论数 100+)
2. 翻评论区每一条,记录:
   - 用户原话吐槽
   - 高频痛点(同类 ≥3 次)
   - 对现有方案的卡点
3. 归类整理 → 痛点频次表
4. 把高频用户原话作为 input 喂 AI:
   - title:必须含用户高频原话词(不是产品语言)
   - description:同样
   - H1 / H2:用用户语言
   - 与 AI 反复拉扯,不要"写一个 SEO friendly 的 title"这种粗暴 prompt
```

**底层洞察(原文金句)**:"AI 把'写代码'这件事变得几乎免费的时候,独立开发者真正的瓶颈,已经从'写得多快'变成了'你到底有没有搞清楚用户要什么'。代码层面我和别人没什么差距,AI 拉平了一切。真正能拉开差距的,是愿不愿意在写代码之前,先花时间去读用户原话。"

### C. 上线后 24 小时迭代节奏

| 时机 | 动作 | 目的 |
|---|---|---|
| 上线 day 0 | 首页 SEO 一遍过 → 提交 GSC | 别纠结细节,**先到位** |
| day 1 | GSC 没反应 → 不慌 | 大部分新站第一天没收录是常态,**别基于"24 小时来流量"建立期待** |
| day 2 流量来了之后 | GA 看用户来源 → 移动端为主立刻优化 | 移动占大头时桌面排版没用 |
| day 2-4 | 看竞品有没有跟进同关键词 → 有就加博客内页 | 内页接住竞品挤过来的溢出流量;选题反推自 Reddit 用户高频问题 |

**铁律**:"完美不重要,**在场才重要**。先上线,再迭代,流量来了再优化移动端,竞品出现了再加内页。"

### D. 多次下注的心理建设

- **数据**:6 站全平,第 7 站起。**前 6 站和第 7 站做的是同一套动作**,差别只是"这次轮到我了"。
- **底层**:不是鸡汤,是 SEO 行业的统计现实——长尾词 + 新站 + 算法波动 + 竞争窗口,**单站起量本就是低概率事件**,需要用站数量博概率。
- **对应 [用数量博概率](../principles.md#用数量博概率做-n-个-v01胜过赌-1-个完美)**:作者用 6-7 个站验证了"AI 让单站建站成本可承受 → 矩阵打法成立"。

## 风险提示

1. **流量 ≠ 收入(公理 4)**:作者全篇讲 146K 展示,**完全没提**:
   - AdSense 是否申请通过 / 通过了多少
   - CTR / CPM / CPC
   - 实际收入数字

   "146K 展示" 是 GSC 数据(搜索结果出现次数),不是 GA 数据(实际访问),**真实流量和真实收入仍未知**。读者要警惕被 GSC 大数字误导,作者作为内容创作者天然倾向报告好看的数字。

2. **样本 = 1**:6 站失败 + 1 站起,"按次数博概率"的具体概率仍未知(可能是 1/7,也可能是 1/30,这次只是早碰到)。复制者可能跑 15 站还没见水花,**心理预算应该拉到"<= 6 站失败是正常起点"**。

3. **未公布关键词品类**:作者全篇没透露具体词是什么、什么 niche。理性(不愿被跟卷),但也意味着读者无法判断:这个起量是**品类红利**还是**方法论生效**。

4. **Reddit 爆贴信号已被很多人盯**:差异化在**快**——评论刚 100+ 就丢 Trends 看,而不是 1000+ 之后再去抢。慢的人吃不到红利。

5. **"day 1 没动静"是经验总结,不是规律**:对已有索引基础的站(同根域 / 同模板批量站),day 1 收录可能更快;对全新域名,可能需要 2-7 天才被爬。**别基于"day 2 应该来流量"建立期待**。

6. **公理扫描**(2026-05-01):
   - 公理 4(流量≠收入)⚠️ — 主指标是展示量,无收入数据,但作者明确承认这是流量阶段成果未夸大,通过(读者需自行警惕)
   - 公理 1(模式独立于人)✅ — 前 6 失败 + 第 7 起,作者明确把差别归因于"按下次数",不归因于个人天赋
   - 准公理 A(形态决定复利)✅ — SEO 内容站是资产型,符合
   - 其余公理无冲突
