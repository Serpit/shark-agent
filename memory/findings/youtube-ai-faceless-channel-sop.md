## 二、核心 SOP 与底层原理

### 1. 五术(决策框架,跨方向通用)

| 步骤    | 内容                    | 底层逻辑 / 关键变量                                      |
| ----- | --------------------- | ------------------------------------------------ |
| 选兴趣   | 只做你"上头"的方向            | 兴趣是续命药——冷启动期没有正反馈,只有兴趣能撑过低谷                      |
| 抄 MVP | 找头部对标,直接抄,不要创新        | "抄生二、二生三",MVP 阶段唯一目标是验证赛道走得通,不是差异化               |
| 具象化需求 | 反复追问"你要的到底是什么",直到不能再细 | 需求颗粒度 = AI 产出质量;模糊需求 → 模糊代码 → 不能用                |
| 自动化提效 | 用 AI 编程把每个工序压到 1 分钟以内 | 提效不是为了更快,而是为了**复制扩张**——一个号能跑通后,边际成本接近 0 才能开 N 个号 |
| 成本核算  | 时间成本 + 经营成本同时算        | 时间是绝对稀缺,先算时间成本反推优化方向;经营成本决定能开几个账号、能否雇人           |

外加心法:**"先把东西发出来,然后不停迭代"**——糙就糙,先有 v0.1,再谈优化。

### 2. YouTube 中文 AI 内容打法(具体可复用)

- **入口选题**:佛学、预言、基督教这类"宗教 + 危言耸听" 内容是 YouTube 中文区高播放赛道(作者三个 YPP 账号都打这三个细分)。
- **找对标的方法**:在 YouTube 多刷类似账号,平台会推荐更多同类型;关注几个就能源源不断挖到生态里的账号。
- **MVP 内容形态**:AI 文本(伪原创对标稿) + AI 图片轮播(MJ 生成) + AI 配音(ttsmaker 起步,后期 ElevenLabs 类) + 剪映拼接。**全程 0 出镜 0 出声**。
- **自动化栈**:Python + AI 编程(Cursor/Windsurf)。作者强调"不会一行代码"也能做——核心是**学会向 AI 描述具象化需求**,而不是学语法。
- **扩张节奏**:第 1 个号跑通(11 月)→ 用 1 个月做提效脚本 + 成本核算 → 第 2、3 号跟进(12 月)。**不是验证立刻铺号,而是先把工业化能力建起来再铺**。

### 3. "具象化需求"的范式(本文最具迁移价值的方法论)

作者举了"图片配文本剪辑"的具象化过程,可抽象成模板:

```
模糊需求 → 列出所有逻辑漏洞 → 把每个漏洞用规则填实 → 得到可执行规范
例:
"图片要合适地展示" 
→ 漏洞:多长才合适?切换发生在哪里?转场用什么?
→ 规则:每张图最少 5 秒,在 5 秒后第一个字幕断点切换,转场从渐显/缩放/弹跳/雨刷四选一,持续 0.7 秒
```

这套"漏洞驱动具象化"的提问范式可直接用到任何 AI 编程 / 产品需求场景。

## 三、风险提示

1. **赛道时效性 / 红利已过**:作者文中数据是 2024-11 至 2025-02 期间。到 2026-04(本次摄入时间),YouTube 中文 AI 内容赛道已是红海;同期作者后续连载也透露**自己 50 万粉的两个高级 YPP 账号被封号** → 平台对 AI 低质内容的打压已经显著加强。**不要假设"13 天开 YPP"今天还能复刻**,要把这条 SOP 当作"验证过的打法骨架",而非"现在抄就赚"的捷径。

2. **平台依赖风险极高**:YPP 政策、Google 对 AI 内容的检测、单账号被封会归零所有积累。如果跑这条路,必须从 day 1 就做账号矩阵 + 内容资产沉淀(脚本 / 素材库可跨平台复用)。

3. **"AI 编程是工业革命,现在不上车就晚了"叙事带焦虑营销味**:观点底层成立(AI 确实降低了开发门槛),但"iPhone 4 发布前夜买苹果股票"这种类比是销售话术,不是判断依据。提取方法论,但别被叙事节奏带着走。

4. **成本数据偏低估**:文中 3-5 RMB / 视频只算了 API 调用,**没算账号养护、被封号的沉没成本、自己的时间机会成本**。真要做,自己核算时把这几项加进去。

5. **作者样本偏差**:作者承认自己有过 23 年 AI 知识付费 7 万 +、24 年红包封面 3 万 +、母婴号 20 万 + 等经历——已经是连续做出小成的"老手"。普通人 0 经验照抄五术框架,可能在"具象化需求"和"先发再迭代"两步都需要更长爬坡时间。
