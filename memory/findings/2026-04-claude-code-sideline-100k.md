## 二、核心 SOP / 原理

### A. 交付半径理论(Claude Code 的真正价值)

**核心论断**:Claude Code 不是补全工具,是工程协作工具——通读项目、理解依赖、跨前后端 + DB + Docker 完成完整任务。它把**单人交付半径**从"个人开发者"拉到"小团队"。

**作者搭的固定工作流**:
- **Skill**:常用能力沉淀成可复用模块(部署 / SEO 审计 / 站点初始化)
- **Agent**:多步骤任务自动化(建站 / 抓取 / 清洗 / 发布)
- **Hook**:提交后自动跑格式化、检查、安全扫描
- **凭证管理**:项目按名称分文件夹,部署脚本读环境变量

**产能数据**:
- 数据看板(FastAPI + Next.js + PostgreSQL + Docker Compose)——2-3 天 MVP
- 自动化工具——1 周可跑
- SEO 站点(选词 → 建站 → 上线)——一个下午

**底层机制**:不是"一个人变成全栈团队",而是"过去需要团队协作的工程步骤,被拆成 AI 可稳定执行的流程"。这个观点解释了为什么 Skill / Agent / Hook 的组合比单纯的代码补全 ROI 高一个数量级。

### B. 抖音技术图文红利打法(陌生平台找空白格式)

**判断逻辑**:技术内容在抖音上大多是视频,**图文反而很空**——找空白格式而不是和大流量内容卷。

**操作 SOP**:
1. **内容形态**:截图 + 简短说明 + 真实过程,1-2 小时一篇
2. **选题原则**:不是教程合集、不是标题党,就是"今天用 Claude Code 做了什么 / 遇到什么问题 / 怎么拆任务 / 跑出什么效果"
3. **冷启动数据**:9 篇内容 → 3 篇 10w+ → 一周千粉 → 不到两千粉吸引 7-8 个企业客资

**关键反思(作者主动停更的原因)**:
- 抖音流量大但**人群杂下沉**,7-8 个客资里只有 1 个真匹配
- 平台流量 ≠ 精准客资;**账号定位不够精准会消磨成本**(沟通时间 / 心力)
- 高客单非标项目本身限制复利:"我不想继续做定制化"

**对用户的启发**:陌生平台的"格式空白"逻辑可平移到 X / Reddit——用户当前 X 内容支柱里"独立开发出海 / 海外 App 商业化拆解"在中文 X 也是相对少见的格式。

### C. SEO 批量建站完整流程(本文最可复用的资产型 SOP)

**技术栈**:Astro + Tailwind CSS + Cloudflare Pages,模板跑顺后单站 2-3 小时上线。

**完整流程**:

| 步骤 | 工具 / 动作 | 关键原则 |
|---|---|---|
| 选词 | Ahrefs 筛选**有搜索量、低竞争**关键词 | **新词最有机会**——最近才出现搜索量的词通常竞争最小 |
| 建站 | Astro / Tailwind / Cloudflare Pages 模板化 | 模板跑顺后单站 2-3 小时,可批量化 |
| 内容 | AI 生成初版 + **人工校验** | **不能完全放手**——内容不准流量起来也留不住 |
| 内页 + 外链 | 持续扩长尾页面 + 高质量外链 | 提升站点权重 |
| 变现 | Google AdSense(手动核心广告位 + 自动广告补充) | 当前月几十美金,在涨 |

**作者实测**:二三十个站,目前月广告收入"几十美金"。

**核心论断**:"教培是不上课就没收入,SEO 是睡觉时也产生收入"——**资产型副业 vs 卖时间副业的形态差异**,即使金额还小,也是真正的复利起点。

### D. 半自动化外链系统(本文最值钱的方法论)

**核心组件**:Claude Code + Playwright MCP + PostgreSQL 平台流程库

**作者最关键的洞察(本文金句)**:
> "很多人以为工具的价值在代码,但真正值钱的是流程数据。代码可以重写,平台经验和失败记录很难凭空生成。这完全类似于为什么工业软件值钱的逻辑。"

**PostgreSQL 流程库的核心字段**:平台类型 / 提交入口 / 注册方式 / 审核规则 / 注意事项 / 成功率 / 失败原因(几千个平台条目)。

**完整流程**:
1. 从 Ahrefs 导出**竞品外链**数据(成功率最高的来源)
2. 导入系统,过滤 spam、低质站点和已提交记录
3. AI 辅助判断目标域名质量
4. **人工确认**一批可做目标(关键节点必须人把关)
5. Playwright MCP 半自动执行注册、填写、提交
6. 系统自动回查外链是否生效

**实测数据**:
- 提交成功率约 30%(10 个站 3-4 个成)
- 6 个新站 2 周提交 300+ 外链
- DR 从 0 突破 40,最高 50+

**作者的反向经验**:
- **全自动不可行**——每个站点流程差异太大,纯脚本覆盖不了。稳定方式是"AI 判断 + 执行辅助 + 人把关关键节点"。
- **博客评论成功率低、spam 比例高,放弃**;竞品外链是 ROI 最高来源。

### E. Skill 包产品化(eat your own dog food)

**触发**:作者把外链系统方法论 + Skill 包 + 平台流程库打包,在社群里有人直接问能不能买,卖了几千块。

**核心论断**:
> "不一定非要做 SaaS。只要把一套高频、痛苦、可复用的流程整理出来,它就可能是一个小产品。"
> "eat your own dog food 是最好的产品 idea 模式——你在解决自己真实的痛点,完全清楚卖点。"

**对用户的启发**:用户当前的 BRD 调研 SOP / 个人 IP 内容工作流 / shark-agent 自身,理论上都可以走这条产品化路径——前提是先内部跑通,再考虑外卖。**但要注意**用户阶段不同(M1 仍在 BRD,作者已 4 个月迭代),目前不必急于打包,先把方法跑顺。

### F. 副业组合的三类资产分层

| 类型 | 代表 | 优势 | 劣势 |
|---|---|---|---|
| **一次性高客单交付** | 企业定制项目 7w+ | 现金流大、验证交付能力 | 非标 / 客户需求会变 / 消磨灵魂 / 不复利 |
| **资产型** | SEO 站矩阵 + AdSense | 长期复利 / 睡后收入 / 系统化 | 起量慢(目前月几十美金)/ 平台依赖 |
| **产品化(经验打包)** | Skill 包 + 平台流程库 | 痛点清晰 / 高毛利 / 可复用 | 受众窄 / 需先内部跑通 |

**作者的取舍轨迹**:停抖音(高客单非标) → 主推 SEO + 工具产品(资产 + 产品化)。**这个取舍方向与用户 profile 中"长期维护必须匹配订阅或持续收入,不能变成无偿客服"完全一致**。

## 三、风险提示

### 适用边界

- **作者背景的隐性 cost**:NUS 数据科学 + 主业跨境电商 + 全栈工程能力 + 2 个 Claude Max Plan(月成本 $400)——单"两三天交付数据看板"这个动作就有大量前置积累,新手不能按这个产能预期。
- **抖音红利窗口性**:作者明确这个红利来自"抖音当时强推图文 + 技术品类视频饱和"的组合。**这个红利窗口可能正在关闭**——后续技术图文供给增加 + 平台推流策略变化都会改变冷启动曲线。**不要把"9 篇过万"作为今天进抖音的预期**。
- **SEO 资产成型周期长**:文章读起来像"30 站月入几十美金"是正常 SEO 节奏,但**容易被读者误读为"很快就月入几十美金"**——实际这是 4 个月持续投入(选词 + 建站 + 外链)的累积结果,且作者强调每月在涨,意思是"现在还小"。**预期 SEO 站到月入 1k 美金量级,9-18 个月是常见节奏**。
- **企业项目"消磨灵魂"是真信号**:7w 看着多,但折时薪 + 沟通成本 + 客户需求变更 + 持续 1-2 个月维护,实际不轻松。**作者主动放弃这条路是策略性选择,而不是"7w 不够"**——这个判断应该被尊重,而不是被读者反向理解为"我也能拿这个 7w 单子"。

### 与已知风险模式的对照(参照 [risks.md](../risks.md))

- **"AI 编程焦虑营销"**(已记录):**本文不踩**——作者反而开篇就说"这不是 AI 一键赚钱的故事,中间有大量试错、沟通、返工和脏活累活",叙事克制扎实。
- **"数字模板被动收入幻觉"**(已记录):**轻微触边但不严重**——作者 Skill 包"几千块"是真实小数额,且明确强调企业项目仍是大头,没有用"被动收入"包装。但读者切片传播时,"4 个月 10w"标题可能被去语境化,**警惕标题党流变**。
- **"独立开发者低价定价陷阱"**(已记录):**本文不踩**——SEO + AdSense 是流量型变现不是直接定价,Skill 包社群内出价合理。

### 新发现的风险模式雏形(单证据,暂不入库)

- **"高客单非标交付反向消耗复利节奏"**:抖音→企业 7w 项目看似 ROI 高,实际占用作者 1-2 个月持续交付时间,期间无法投入 SEO/工具/IP 三条复利线。这是单证据,但与既有 principles "副业要做能离开人也跑的事"(下文新增)一致。**等到再有 2-3 个独立证据时,可考虑提升为正式 risks 条目**。

### 个人入场建议(若用户参考本文)

- **不要直接复刻抖音技术图文路径**——M1 阶段精力应锁定 X(已是 [experiments 个人 IP 实验](../experiments.md#个人-ip-定位与内容主线验证) 主战场),抖音引流到企业大单与"复利型副业"目标背道而驰。
- **可优先吸收的 3 件事**:
  1. **Claude Code 工程化范式(Skill / Agent / Hook)** 直接套用到 shark-agent 自身 + 未来 MVP 工具产品
  2. **SEO + AdSense 内容站打法** 作为"资产型副业"候选评估,可放进 themes.md 候选池(下方已回流)
  3. **"流程数据 > 代码"原则** 作为产品化方法论吸收到 principles(下方已回流)
- **暂不优先吸收**:抖音冷启动、企业大单交付——前者红利窗口不稳,后者与"避免无偿客服"profile 偏好冲突。
- **关注作者后续输出**:作者明确说会在 X 上"持续记录真实过程",可作为 SEO 出海实战的长期跟踪样本。
