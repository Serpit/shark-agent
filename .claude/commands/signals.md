---

把用户这次输入的风向标作为一条信号摄入并评估,产出三段式判定,落到 `memory/signals/`。

**输入**：`$ARGUMENTS` 是用户从生财社群复制过来的风向标原文。如果为空,回复「风向标呢？在 /signals 后面把内容贴进来」并停下,不要新建空条目。

## 步骤

### 1. 查重
对 `$ARGUMENTS` trim 后计算 sha256 前 12 位,在 `memory/signals/_processed.md` 中查找。命中则告知用户「这条风向标已评估过,见 <signal 链接>」并停下。

```bash
printf %s "<原文>" | shasum -a 256 | cut -c1-12
```

### 2. 读权重与风险库
评估前必须先读:
- `memory/profile.md` — 拿到适配度判定的权重
- `memory/risks.md` — 拿到已知风险模式,评估真伪要对照
- `memory/timeline.md` — 判断时机是否匹配当前阶段

### 3. 写 signal 文件
在 `memory/signals/` 下用 `YYYY-MM-DD-<英文-slug>.md` 命名(slug 从风向标主题提炼,3–5 个连字符词)。文件内容严格按下面三段式,**不要复述原文**(原文存档放最末尾的「原文备份」区块即可):

```markdown
# <一句话主题>

> 摄入时间:YYYY-MM-DD HH:MM (UTC+8)
> 信号源:<作者/账号,如果原文有>

## 1. 适配度评估
**判定**:高 / 中 / 低 适配 (选一个)

按 `profile.md` 的四个权重维度逐条给理由(出海主线、需求挖掘、试错容忍度、人脉信号),没命中的维度也要明确写「不相关」,不要省略。

## 2. 市场逻辑
**判定**:成立 / 部分成立 / 不成立

至少回答这四点,缺哪一项就标注「信息不足」:
- **需求真实性**:谁在为什么场景买单?是真痛点还是制造焦虑?
- **规模与可持续性**:天花板在哪,是一次性套利还是长期生意?
- **竞争与护城河**:门槛是什么,先发是否重要,容易被复制吗?
- **关键变量**:决定成败的 1–2 个变量是什么(平台规则、流量来源、客单价等)?

## 3. 真伪与风险
**判定**:可信 / 存疑 / 疑似炒作

对照 `risks.md` 已知模式给依据。如果发现新风险模式,本次同步追加到 `risks.md` 并在这里链接过去。

## 行动建议
一句话:**深入研究** / **持续观察** / **不跟**,以及给出理由。

---

## 原文备份
<原样粘贴 $ARGUMENTS>
```

### 4. 回流到聚合视图
- 行动建议是「深入研究」或「持续观察」,且支持/反驳某个出海方向 → 更新 `themes.md` 对应条目的证据链接。
- 揭示新风险模式 → 已在第 3 步写进 `risks.md`,确认链接回到 signal 文件。
- 提到值得连接的人或社群 → 追加到 `connections.md`。
- **不要**新增 principles——风向标偏机会侧,跨方向方法论从 findings 里沉淀,这里只做机会评估。

### 5. 登记入清单
在 `memory/signals/_processed.md` 追加一行(摄入时间用 UTC+8,主题摘要 ≤ 20 字,哈希前 12 位,适配度,真伪判定,signal 链接)。如果文件不存在,先创建表头(见下方模板)。

### 6. 同步索引
确认 `memory/INDEX.md` 的「信号摄入」入口仍指向 `signals/`,缺则补上。

## _processed.md 模板

```markdown
# 已评估风向标清单

防重的真相之源。每次评估新风向标前,先查 sha 是否已存在;评估完成后追加一行。

| 摄入时间(UTC+8) | 主题摘要 | sha256(前 12 位) | 适配度 | 真伪判定 | signal 链接 |
|---|---|---|---|---|---|

> 适配度:`高` / `中` / `低`
> 真伪判定:`可信` / `存疑` / `疑似炒作`
```

## 输出回复

写完所有文件后,只回三行:
1. 一句话判定(主题 + 适配度 + 真伪 + 行动建议)
2. signal 文件路径
3. 涉及哪些聚合视图被更新(没更新就写「无聚合视图变动」)

不要在终端输出里复述三段式内容——用户自己会去看 markdown 文件。
