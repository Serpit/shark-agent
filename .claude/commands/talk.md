把用户输入作为教练咨询的起点。支持两种模式：**问诊**（用户带具体问题来）和**体检**（用户没有明确问题，做系统性排查）。

**输入**：`$ARGUMENTS` 为用户的问题描述。如果为空且用户首条消息含问题意图，用首条消息作为输入；如果用户只说 `/talk` 无其他内容，进入模式选择。

## 步骤

### 0. 模式选择

若用户没提供具体问题，先问：
```
想怎么聊？
A. 问诊 — 带一个具体问题来，我帮你拆
B. 体检 — 跑一次系统性排查，看看卡在哪
```

用户选 A → 等用户输入问题后进入步骤 1。
用户选 B → 跳过问题输入，直接从「当前阶段目标 + 最近 3 条 talks 未结转 + running 实验状态」启动排查，公理扫描 9 条全量过一遍，找出潜在卡点。

### 1. 问题分类

读用户输入后，先用公理 1/3/6 做问题分类：

| 分类 | 特征 | 处理方式 |
|---|---|---|
| **信息问题** | 缺具体事实/数据/操作知识,答案可被检索 | 直接回答 + 给信息来源 |
| **模式问题** | 方向/策略/定价/形态选择 | 公理扫描 → 对照 principles/risks → 给判定和选项 |
| **心理卡点** | 有足够信息但迟迟不行动,理由循环 | 触发公理 6,追问"信息还是心理?" |
| **假问题** | 问题前提错误(如"该选 A 还是 B"但都不该选) | 消解问题:指出前提错误 + 给第三选项 |

### 2. 公理扫描

无论哪种分类,都必须跑一次 9 条公理扫描。用 [memory/methods/axiom-scan.md](../memory/methods/axiom-scan.md) 的逐项问句,触发哪几条在对话中标注。

- 第一层公理 1-6 触发 → `⚠️ 公理 X 触发：<原因>`
- 第二层准公理 A/B/C 触发 → `⚡ 准公理 X 追问：<具体问句>`

### 3. 关键追问

公理扫描触发的追问逐条抛出,一次最多 3 个。等用户回答后再进入下一步。

### 4. 消解或回答

- **假问题** → 消解：指出前提错误 + 重新定义问题 + 给第三选项
- **心理卡点** → 先问"这是信息问题还是心理问题?"。用户承认是心理问题 → 追问"最小可执行动作是什么?"(把大象拆成一口)。用户否认 → 按信息/模式问题处理,标记"下次同类循环触发公理 6"
- **信息问题** → 直接回答,给信息来源和验证方式
- **模式问题** → 对照 principles/risks 给判定(高/中/低适配) + 明确行动建议(做/不做/深入调研)

### 5. 落到行动

把输出收束成具体下一步:
- 若涉及立项/实验 → 写入或更新 `memory/experiments.md`
- 若为一次性动作 → 写入 `memory/todos.md`
- 若为信息缺口 → 生成 `todo` 标注"需调研 X"

### 6. 写 talks 文件

在 `memory/talks/` 下使用 `YYYY-MM-DD-<topic-slug>.md` 命名,按 [memory/talks/README.md](../memory/talks/README.md) 的 6 段模板写入。

### 7. 回写相关记忆

- 若得到新认知/方法论 → 更新或新增 `principles.md` 条目
- 若暴露新风险模式 → 追加 `risks.md`
- 若关联到现有实验 → 在 `experiments.md` 对应实验下追加「talk 关联」行

## 输出格式

对话中给用户看：
1. 分类判定（一行）
2. 公理扫描触发项（逐条）
3. 关键追问（如有）
4. 消解/回答 + 行动建议
5. talks 文件路径

对话结束后确认文件已写入。
